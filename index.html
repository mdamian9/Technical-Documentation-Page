Introduction

Greetings everyone! This is a very quick introduction to getting started with GitHub.

This documentation will go over the very basic steps to creating a project on GitHub, making commits, and deploying to GitHub pages.

In the next sections, we will go over the following:

  • Creating New Repositories
  • Importing Existing Repositories
  • Making commits to GitHub
  • GitHub Desktop
  • GitHub Deployment

New Repositories

Once you've created a new account on GitHub, you're ready to create your first project.
In order to do this, you can click on a button that is found in your account home page. It's a green button that says "New Repository."
You'll be able to create a new repository by clicking this button.

After you've clicked this button you'll now see the following on your screen. Follow the instructions provided.
You can name your project and provide a description. You can choose whether your repository is public or private.
If you select public, anyone can see your code, but you choose who can make edits to it. If you choose private, only you can see and edit your code.
If you'd like to add a README file to your project, you can check the box. Click on "create repository" and you're ready to go.

If you did not check the box to include a README, you will now see the following screen.
Follow the instructions for "... or create a new repository on the command line"
Open a terminal window and navigate to the root folder of your project and type in the following:

git init
git add README.md
git commit -m "first commit"
git remote add origin [your repository url goes here]
git push origin master

If you did check the box to include a README, you will see the following screen.
You're now ready to clone your project to your machine and start making commits.
To clone your project to your machine, simply open a new terminal window and type the following command:

git clone /path/to/repository

After you've modified your project in a code editor, you can open up a terminal window, and navigate to the root folder of the project.
To make a new commit, enter the following in your terminal:

git add .
git commit -m "[message describing the commit]"
git push origin master

You have now made a new commit and your project on GitHub will be updated with the new code.


Existing Repositories

If you already have initialized a repository on your local machine, simply do the following:
- Create a new repository on GitHub as described by the steps above.

git remote add origin [your repository url goes here]
git push origin master

GitHub Deployment

To deploy to GitHub Pages and make your site available to everyone around the world, you first need to click on your repository, and click on the settings button.

After clicking on the settings button, scroll down to where it says "GitHub Pages" and click on the dropdown menu.
Select "master branch" and click on save. Your web application should now be live on the internet within the next couple of minutes.


GitHub Desktop

There is a version of GitHub that is available as a desktop program or application. It is available for download on the GitHub site.

The desktop is very convenient for users that would like to keep from having to open up a browser every time to sign in to GitHub.

Users can make commits and make pull requests from within GitHub desktop, and the application offers many other great features.


Conclusion

In conclusion, you now know how to create your first repository on GitHub.

You know how to create a new repository from scratch, and you know how to import an existing repository from your machine to GitHub.

You've also learned how to make commits, and how to deploy your web application to GitHub Pages.

You've learned about GitHub Desktop and some of its features. Always use the following command to keep track of your files in your project.

git status
